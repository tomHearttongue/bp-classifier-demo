This section explains the production pattern behind the prototype. It is not part of the playable workflow; it shows how the orchestration layer would select the right tool for each part of the support process.

Request contextUser, role, location, department, and service history.

AI-Assisted Layer

              

                
Intent classificationUnderstand the employee's natural-language need.

                
Grounded retrievalFind approved policy, procedure, and service metadata.

                
Handoff draftingSummarize context for the human reviewer.

              

            

            

              
Deterministic Controls

              

                
EligibilityRole, approval, access, and policy checks.

                
RoutingQueue selection, SLA class, and exception handling.

                
ActionsTicket creation, workflow state changes, and audit logs.

Reliability Rule

              
No approved source means no authoritative answer. The system can collect context and route the gap, but it should not invent policy.
